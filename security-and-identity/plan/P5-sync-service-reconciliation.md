## Problem statement

Harden the basic KAOS->broker sync service from P3 into an MVP-robust reconciler: it must reliably project Agent / MCPServer / **ModelAPI** topology into broker records and per-agent credential Secrets, and — the headline gap — **reconcile deletions** so that when a KAOS resource is removed the broker records (agents, permission sets, synthetic services) and credential Secrets it owned are pruned, with no orphans. It must report drift/status (counts, per-agent state, categorized problems) through logs, Prometheus metrics, and health endpoints, and never silently degrade: transient broker failures are retried with backoff, a single bad resource never aborts the whole pass, and broker-unreachable surfaces as not-ready rather than a silent no-op. It stays a standalone Python service in the KAOS repo (extraction is later, P10); active credential rotation is also later (P10).

## Current-state grounding (researched)

The sync service lives at `sync-service/` (Python package `kaos_sync`, `Dockerfile`, `Makefile`, Helm `chart/`, `pyproject.toml`, `tests/`). Modules:

- **`projection.py`** (pure, I/O-free): `project(resources) -> DesiredState`. Encodes only **MCPServer** edges today: `DesiredService` (`client_id` `kaos-mcpserver-<ns>-<name>`, single `call` scope), `DesiredPermissionSet` (`kaos:mcpserver:<ns>:<mcp>:call`), `DesiredAgent` (`external_id` `kaos://agent/<ns>/<name>`, bound permission sets). Helper fns `service_client_id`, `permission_set_name`, `agent_external_id`, `mcpserver_resource_uri`. Agents with no edges are skipped. **`spec.modelAPI` is NOT projected.**
- **`reconcile.py`** (I/O-free; orchestrates `AIBAdminClient` + `SecretStore` protocols): `reconcile(desired, aib, secrets, secret_prefix) -> ReconcileSummary`. Create-or-get services -> permission sets -> agents; mint credentials only when the Secret lacks a `client_id` (idempotent, no churn). `credential_secret_name(prefix, name) = "<prefix>-<name>"`. **No deletion/pruning, no drift detection, no retry.**
- **`aib_client.py`**: `AIBAdmin` over the broker admin REST API (pre-auth `X-Remote-User` header). Methods: `create_or_get(collection, match_field, match_value, body)` (POST then list-scan fallback), `mint_credentials(agent_id)`. **No `list`, `delete`, or `get`-by-id exposed for pruning; no retry/backoff.**
- **`k8s.py`** (runtime only): `KubeSecretStore` (get/upsert Opaque Secrets, labels `app.kubernetes.io/managed-by: kaos-sync`; **no delete, no list**), `KaosResourceLister` (lists `agents` + `mcpservers`; **not `modelapis`**). `load_kube_config()`.
- **`config.py`**: `Settings` dataclass from env (`AIB_ADMIN_URL`, `AIB_PRINCIPAL`, `AIB_PRINCIPAL_HEADER`, `KAOS_SYNC_NAMESPACES`, `KAOS_SYNC_CREDENTIAL_SECRET_PREFIX`, `KAOS_SYNC_RECONCILE_INTERVAL_SECONDS`, `KAOS_SYNC_REQUEST_TIMEOUT_SECONDS`). No metrics/health/prune/backoff settings.
- **`main.py`**: `run_once(settings, lister, aib, secrets)` + a `while True` loop with one broad `try/except` around the whole pass and a single summary log line. **No metrics server, no health endpoint, no readiness gating, no per-resource isolation.**
- **`chart/`**: `deployment.yaml`, `rbac.yaml` (ClusterRole: read `agents,mcpservers,modelapis`; secrets `get,list,watch,create,update,patch` — **no `delete`**), `serviceaccount.yaml`, `values.yaml`, `_helpers.tpl`.

**Broker admin DELETE support (confirmed, enables pruning)**: `internal/adapters/http/routing/admin.go` exposes `DELETE /api/agents/{id}`, `DELETE /api/services/{id}`, `DELETE /api/permission-sets/{id}`, and `DELETE /api/agents/{id}/client-credentials` (revoke). So orphaned records can be deleted via the admin API.

**Edge scope**: the Agent CRD (`operator/api/v1alpha1/agent_types.go`) has `spec.modelAPI` (single string) and `spec.mcpServers` ([]string). There is no static sub-agent/A2A edge list field, so Agent->Agent edges are out of scope for projection (delegation is runtime, not a CRD edge). P5 adds the ModelAPI edge; Agent->Agent stays out.

**Target (ADR-KAOS-008 sync responsibilities)**: discovery (Agent/MCPServer/ModelAPI), identity projection on stable `external_id` preserved across delete/recreate, requested-edge projection (kept distinct from approved grants), bootstrap synthetic-service/PermissionSet encoding, per-agent credential provisioning into Secrets, and **status/drift reporting** (report broker-unreachable, record drift, missing credentials, stale external IDs, unsupported requested edges; never silently fall back to insecure behavior).

## Design plan (how it fits)

Keep the clean split: pure `projection.py`/`reconcile.py` (protocol-driven, fully unit-tested with fakes) and thin runtime adapters in `k8s.py`/`aib_client.py`/`main.py`. Additions:

- **Pruning (delete reconciliation).** Reconcile becomes a full desired-vs-actual diff for the records the sync owns. Ownership is determined by the existing **`kaos-` / `kaos:` naming convention** on broker records (services `kaos-*`, permission sets `kaos:*`, agents whose `external_id`/`display_name` starts `kaos://agent/`) and the **`app.kubernetes.io/managed-by: kaos-sync`** label on Secrets. After applying the desired state, list owned broker records + owned Secrets and `delete` any not present in the desired set. Pruning is **safe-by-construction**: only records matching the kaos ownership markers are ever deleted; never touch foreign records. Order matters: delete agents (and revoke their credentials + delete their Secret) before permission sets before services, to respect referential dependencies. A `prune_enabled` setting (default `true`) gates it; when off the service is create/update-only (P3 behavior).
- **ModelAPI edge projection.** Extend `projection.py` with `kaos-modelapi-<ns>-<name>` synthetic services, `kaos:modelapi:<ns>:<name>:call` permission sets, and `modelapi_resource_uri(ns,name) = kaos://modelapi/<ns>/<name>`. An Agent's `spec.modelAPI` yields one ModelAPI edge bound to the agent (alongside its MCPServer edges). Generalize the existing MCPServer helpers to a shared `_synthetic_service`/`_edge` shape keyed by resource kind to avoid duplication. An agent with neither MCPServer nor ModelAPI edges is still skipped.
- **Status/drift reporting + observability.** `reconcile` returns a richer summary: per-agent status, prune counts, and a list of categorized problems (`AIB_UNREACHABLE`, `MISSING_CREDENTIALS`, `UNSUPPORTED_EDGE`, `STALE_EXTERNAL_ID`, `PRUNE_FAILED`). `main.py` logs structured status and updates Prometheus counters/gauges exposed on a `/metrics` HTTP endpoint (use `prometheus_client`), and serves `/healthz` (process up) and `/readyz` (last pass succeeded and broker reachable). Optional: annotate the KAOS resource with a last-sync status (stretch; only if RBAC `patch` on the CRD status is added). MVP target = logs + metrics + health; annotations are stretch.
- **Robustness / fail-closed.** Add retry-with-exponential-backoff (bounded) to `AIBAdmin` for transient connection/5xx errors (configurable attempts/base delay). Make the pass per-resource isolated: catch and record a problem per agent/edge so one failure does not abort the rest, while still surfacing it in status/metrics and flipping readiness when broker-wide failures occur. Never silently succeed on partial failure — readiness reflects the last pass.
- **Chart/runtime wiring.** `config.py` gains `prune_enabled`, `metrics_port`, `health_port`, `retry_max_attempts`, `retry_base_delay_seconds`. `aib_client.py` gains `list(collection)`, `delete(collection, id)`, and uses retry. `k8s.py` `KubeSecretStore` gains `list(namespace, label_selector)` and `delete(namespace, name)`; `KaosResourceLister` gains a `MODELAPI_PLURAL = "modelapis"` constant and lists it in `list_resources` (the chart RBAC already grants read on `modelapis`, so no RBAC change is needed for discovery). Chart: RBAC adds `delete` on secrets; Deployment exposes metrics/health ports + readiness/liveness probes; `values.yaml` surfaces the new settings; optional metrics `Service`.
