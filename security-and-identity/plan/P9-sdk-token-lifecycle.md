## Problem statement

Make the AIB SDK production-grade so agents authenticate as themselves without a static, pre-minted token. Today the SDK (`pydantic-ai-server/aib/`) only **propagates** identities: it forwards the inbound user subject `Authorization` and injects a *static* agent actor token read from `AIB_ACTOR_TOKEN` / `security_actor_token` (the P1 simulation contract). That static token cannot expire-safely, cannot survive credential rotation, and provides no recovery on a gateway `401`. P9 implements the **machine actor-token lifecycle** from ADR-AIB-001 / ADR-KAOS-006: the SDK acquires the actor token itself via a `client_credentials` grant against AIB, caches it with refresh-ahead, reloads the underlying credential on rotation without a restart, retries once on a `401`, backs off on AIB unavailability, and **fails closed**. It also adds the optional, off-gateway `Client`/`AsyncClient` surface (`check_access`/`require_access`/`get_token`/`exchange_token`). The runtime is repointed at this lifecycle. The SDK's relocation to AIB's package home and upstreaming are P14, not here.

## Current-state grounding (researched)

- **SDK shape** (`pydantic-ai-server/aib/`): only `__init__.py` (exports `ctx`, `current`, `security_context`, `to_headers`, `instrument_fastapi`, `instrument_httpx`, header constants) and `instrument.py`. `instrument.py` holds: the `ContextVar`-backed request-local `ctx`, `to_headers()` (Bearer-wraps `subject_token`/`actor_token`, additive-merge only), inbound ASGI middleware `_PropagationMiddleware` + `instrument_fastapi(app, actor, actor_token, principal, principal_resolver)` (local identity falls back to `AIB_ACTOR`/`AIB_ACTOR_TOKEN`/`AIB_PRINCIPAL` env), and outbound `instrument_httpx()` which monkey-patches `httpx.Client.send`/`httpx.AsyncClient.send` to call `_inject_request_headers(request)` (additive). **No token acquisition, no refresh, no reload, no retry, no `Client` surface.** `py.typed` present (typed package).
- **Runtime wiring** (`pydantic-ai-server/pais/server.py:141-148`): `local_actor = settings.security_actor or f"kaos://agent/{settings.agent_name}"`; `aib.instrument_fastapi(app, actor=local_actor, actor_token=settings.security_actor_token or None, principal=settings.security_principal or None)`; then `aib.instrument_httpx()`. `security_context=aib.security_context()` is attached to runs (`server.py:400`). Settings (`pais/serverutils.py:427-430`): `security_actor`, `security_actor_token`, `security_principal` (all `str = ""`, env-bound via `AgentServerSettings(BaseSettings)`). So today the actor token is whatever static value `security_actor_token` carries.
- **Operator-provided env** (`operator/controllers/agent_controller.go`, `buildAgentAuthEnvVars`, gated by `security.GetConfig().CredentialMountingEnabled()`): `AIB_ACTOR` (resolved logical id — honours `spec.security.id` after P8), `AIB_CLIENT_ID` + `AIB_CLIENT_SECRET` (from the per-agent credential Secret `<prefix>-<name>` via **`SecretKeyRef` env**, `Optional: true` so the pod starts before the Secret exists), and `AIB_TOKEN_ENDPOINT` (from `cfg.TokenEndpoint()` = `<issuer>/oauth2/token`) when an issuer is set. **Everything needed for `client_credentials` is already in the pod env** — except the credential is delivered as a *static env var*, not a *file*, so rotation does not reach a running pod. ADR-KAOS-009 prefers a projected-volume file mount for exactly this reason; the file mount is **not implemented yet** (only env injection exists). P9 adds it.
- **No `AIB_ISSUER`/`AIB_CLIENT_SECRET_FILE` env today**; the issuer is only materialised as `AIB_TOKEN_ENDPOINT`. The SDK should prefer `AIB_TOKEN_ENDPOINT` directly (no discovery needed) and treat issuer/JWKS as optional.
- **Tests/lint** (`pydantic-ai-server/Makefile`): `make test` = `uv run pytest tests/ -v --cov`; `make lint` = black check + type check (`ty`). Tests live in `pydantic-ai-server/tests/` (e.g. `test_aib_*` propagation tests). Python runtime is async-first (Pydantic AI, httpx AsyncClient); the token manager must be safe for both sync and async send paths.

**Target (ADR-AIB-001 §"Machine identity and actor-token lifecycle" + ADR-KAOS-006)**: `aib.instrument_agent_identity(issuer_env/token_endpoint_env, client_id_env, client_secret_file)` configures a managed actor token; `aib.actor_token()` returns a cached, always-fresh token (`sub`/`azp` = this agent's logical id). The instrumented outbound call becomes the seamless "ensure-fresh + inject + retry" path. Lifecycle: acquire via `client_credentials` against `/oauth2/token` (default `client_secret` auth; `private_key_jwt` left as a future option); refresh-ahead at ~75–80% TTL so the request path never blocks; file-watched credential reload honouring AIB's previous-secret grace window; single reactive `401` retry; backoff on AIB unavailability; fail closed. The `check_access`/`require_access`/`get_token`/`exchange_token` client methods are **optional** (off-gateway custom servers); the gateway path already does authz + token exchange (P2/P7).

## Design plan (how it fits)

### SDK: actor-token manager (new `aib/identity.py`)

- `ActorTokenManager` encapsulating: a credential source (file-first via `client_secret_file`, env fallback `AIB_CLIENT_SECRET`), the token endpoint (`AIB_TOKEN_ENDPOINT`, else `<AIB_ISSUER>/oauth2/token`), `client_id` (`AIB_CLIENT_ID`), and cache state (current token + expiry). Methods: `token()` (sync) / `token_async()` (async) returning a fresh token; `force_refresh()`; `invalidate()`. Refresh-ahead: cache the token with `expires_at`; serve from cache until `now >= expires_at - ttl_fraction*lifetime` (default fraction ~0.2, i.e. refresh at ~80%); refresh is single-flighted (a lock so concurrent callers don't stampede) for both sync (`threading.Lock`) and async (`asyncio.Lock`). Backoff: on endpoint failure, retry with bounded exponential backoff up to a short ceiling, then raise `aib.AIBUnavailable` (fail closed — never return a stale/empty token as if valid). Use `httpx` for the grant POST (`grant_type=client_credentials`, `client_id`, `client_secret`, optional `scope`); parse `access_token` + `expires_in`.
- `instrument_agent_identity(*, token_endpoint_env="AIB_TOKEN_ENDPOINT", issuer_env="AIB_ISSUER", client_id_env="AIB_CLIENT_ID", client_secret_file=None, client_secret_env="AIB_CLIENT_SECRET", refresh_fraction=0.2)` constructs and registers a process-global manager. `actor_token()` / `actor_token_async()` module-level accessors delegate to it; when no manager is configured (or no credentials present) they return `None` (so the static-token / simulation path still works unchanged).

### SDK: credential file source + rotation reload

- The credential source reads `client_secret` from `client_secret_file` when set (projected-volume file), else `AIB_CLIENT_SECRET`. It caches the file contents + mtime and **reloads on mtime change** (checked lazily on each acquire/refresh — no background thread needed), so a rotated Secret is picked up on the next refresh without a restart. Honour the grace window implicitly: on a `401`/`invalid_client` during acquire, reload the credential file once and retry (covers the rotation race).

### SDK: reactive 401 retry in outbound instrumentation

- Extend `instrument_httpx()` so that, when a managed identity is active and an instrumented request that carried `x-agent-authorization` receives a `401`, the SDK calls `force_refresh()` and replays the request **once**. Keep it strictly additive and idempotent (only retry once, only when the response is `401`, only for requests the SDK injected the actor token into). Cover both `Client.send` and `AsyncClient.send`. Requests with no managed identity behave exactly as today (no retry).

### SDK: optional off-gateway client surface (new `aib/client.py`)

- `AccessRequest`, `AccessDecision`, `TokenResult` dataclasses (shapes per ADR-AIB-001) and exceptions `AccessDenied`, `ReauthenticationRequired` (carrying `reauth_url` + decision details), `AIBUnavailable`. `Client(base_url=AIB_BASE_URL, ...)` + `AsyncClient` with `check_access(resource, action, ...)`, `require_access(...)` (raises on non-allow), `get_token(resource, scopes)`, `exchange_token(subject_token, audience, scopes)`. These default their `principal`/`actor`/`request_id` from `aib.ctx`. They call the AIB access-check (`/api/access/check`) and token endpoints. Clearly documented as **optional/off-gateway**; the gateway remains the enforcement path. Keep transport `httpx`, sync + async parity.

### Operator: projected-volume credential file mount

- Extend the agent (and, where credentials are mounted, MCPServer) pod construction so that, when `CredentialMountingEnabled()`, the per-agent credential Secret is also mounted as a **projected volume** at a fixed dir (e.g. `/var/run/aib/`), and `AIB_CLIENT_SECRET_FILE=/var/run/aib/client_secret` is exported. Keep the existing `SecretKeyRef` env vars for backward-compat/bootstrapping; the file mount is what enables rotation reload. Optional `Secret` reference (pod starts before the Secret exists) — Kubernetes keeps it `Pending` until present, consistent with the env behaviour.

### Runtime: repoint at the managed lifecycle

- In `server.py` `_setup_aib`, after `instrument_fastapi`, call `aib.instrument_agent_identity(...)` (reading the operator-provided env) and `aib.instrument_httpx()`. The outbound injection uses `aib.actor_token()` when no explicit/static `actor_token` is configured; the static `security_actor_token` remains honoured (simulation/tests). Net effect: in a real cluster the agent now mints + refreshes its own actor token; in tests/sim the static token path is unchanged.
