# P10 — sync service productionisation (learnings)

## What worked

- **Keeping `run_once` idempotent and contract-stable** let every new runtime path (watch, leader election, rotation, write-back) be layered around it without touching the reconcile core. Watch and leader election are pure orchestration in `main.py` (testable with fakes, no cluster); the cluster-only pieces (`KubeWatchSource`, `KubeLeaseBackend`, `KubeStatusWriter`) stay thin and are exercised in the in-cluster block, mirroring how `KubeSecretStore`/`KaosResourceLister` were already split.
- **Transition-based leader callbacks** (fire `on_started_leading` only on a false→true edge, `on_stopped_leading` only on true→false) plus a `_WorkerHandle._guarded_run_pass` that refuses to mint/prune after leadership is lost means a trailing reconcile iteration after a lost lease cannot double-write. This avoided changing `reconcile_loop` and kept the watch tests untouched.
- **Annotation-based status write-back** (not a CRD `.status` subresource) is the right least-coupling choice: the sync service is deliberately not the operator and must not own a status subresource. A JSON merge patch scoped to `metadata.annotations` cannot clobber spec or fight the operator, and 404-tolerance makes it safe against deletes mid-pass. Modelling per-object results as a `ResourceSync` list on the summary (rather than string-matching the flat `problems` list back to objects) made write-back clean and unit-testable.
- **Rotation via re-mint into the same Secret key**, grace-window-safe (write the new secret, let the broker expire the old after its grace, never invalidate-first), composes directly with the P9 SDK file-reload: the SDK re-reads `/var/run/aib/client_secret` on mtime change, so an in-place Secret update is picked up without a pod restart. Stamping the mint time as an annotation (not a Secret data key) keeps the rotation clock off the mounted file surface.

## Gotchas / decisions

- **Inserting `credentials_rotated` into `AgentSync` before `ok`** broke the existing positional `AgentSync(...)` error-return callsites (the new field shifted `ok`/`error`). Fixed by converting those returns to keyword args. Lesson: when adding a dataclass field that existing positional constructors straddle, convert callers to keywords in the same change.
- **Rotation default is `0` (disabled)** until the broker grace-window behaviour is validated end-to-end. A Secret minted before rotation existed carries no stamp and is treated as *due* (establishes a baseline on the first enabled pass); an unparseable stamp is likewise treated as due and never crashes a pass.
- **The sync chart `image.tag` is `""` and falls back to the chart `appVersion`.** So lockstep is achieved by advancing `Chart.yaml` version/appVersion (in `build-helm` at release time and `bump-version` post-release), not by an `image:tag` `sed` in `values.yaml`. The stale `0.1.0` sync chart version was realigned to the current line as part of this.
- **TODO 6 (repo extraction / subtree mirror / CLI consumption of a published external chart) is deferred** to a manual post-P12 step per the phase scope decision — the sync service stays in-repo for now. The packaging (TODO 5) is done so the image/chart are versioned and publishable; only the standalone-repo split is pending.

## Python → Go rewrite

The Python sync service was subsequently rewritten as a Go `controller-runtime` reconciler. The Python implementation had grown its own control-plane infrastructure — resource listing, watch threads, dirty/debounce signalling, leader election, readiness state, Secret create/replace, and bounded HTTP retry loops — to make `run_once` production-grade. The rewrite's central learning is that almost all of that is undifferentiated controller plumbing that the native Go ecosystem already owns, so the durable improvement came from deleting bespoke machinery rather than porting it line for line.

### What worked

- **Single-sentinel reconcile over a per-object controller.** The projection is a whole-world function — agents reference other resources, and `spec.security.id` collision resolution needs every resource of a kind — so `SetupWithManager` watches all three KAOS kinds but funnels every change to one constant `reconcile.Request`. The workqueue coalesces a burst into a single full reconcile, and controller-runtime supplies the watch, informer cache, leader election, periodic resync and requeue-with-backoff for free. This is the Go analogue of the Python watch+debounce, but as configuration rather than hand-written orchestration.
- **Pure projection vs thin apply, preserved and sharpened.** `internal/projection` has no cluster or broker I/O; it owns naming conventions, edge encoding and identity/conflict resolution, and is unit-tested directly. `internal/sync` only lists CRDs, applies the projected state to AIB, writes Secrets and prunes. Go's compile-time types made this boundary more explicit than the Python protocols did, with the broker-shaped admin body produced next to the naming that derives it and handed to a generic HTTP client.
- **Unstructured client over generated CRD types.** Reading KAOS resources via `unstructured.Unstructured` keeps the sync service decoupled from the operator's generated Go types while still giving typed Kubernetes primitives for `Secret` server-side apply. The service does not need the full CRD schema, only the handful of spec fields the projection reads.
- **Idempotency at every external boundary.** AIB resources are upserted through `CreateOrGet(collection, matchField, matchValue, body)` — POST first, then scan the collection for the matching unique field when creation does not return 2xx — credentials are minted only when the Secret is absent or incomplete, Secret writes use server-side apply, and deletes treat missing records as success. This makes the requeue-on-error retry safe to re-run an entire pass.

### Gotchas / decisions

- **Error handling must match blast radius, and the Go port carried over an asymmetry worth revisiting.** Service and permission-set apply fail fast (downstream grants are untrustworthy without them), agent failures are isolated and counted so other agents still converge before the pass returns an error for backoff, and pruning is best-effort and logged. The fail-fast service/permission-set loops mean one poisoned record can head-of-line-block all provisioning; a fail-soft variant is a known candidate refinement.
- **Replayable request bodies for retries.** `retryablehttp` may resend a request, so bodies are marshalled once into bytes and passed as a `bytes.Reader` rather than a one-shot stream; tying retry to a consumable reader would silently send empty bodies on the second attempt.
- **Pruning needs ownership predicates and dependency order.** The Go code only deletes KAOS-shaped broker records and Secrets labelled `app.kubernetes.io/managed-by=kaos-sync`, and removes agents, then Secrets, then permission sets, then services, so an in-flight delete never strands a referenced dependency.

### Carry-forward

- The Secret name contract (`<prefix>-<agent>`) is derivable by both the sync service and the operator, so the prefix is part of the API surface even though the Secret contents are broker-issued; any change must move both sides together.
- `spec.security.id` collision resolution **was** duplicated by construction — the operator gated reconcile on it and the projection mirrored the same oldest-wins rule so both agreed on the winner. The identity-override surface was subsequently removed entirely (logical identity is always `kaos://{kind}/{namespace}/{name}`), and both copies and their tests collapsed away together — confirming the predicted simplification.
- The opt-in AIB end-to-end credential test (sync creates the Secret → operator mounts it into the agent Deployment → pod starts authenticated) remains the only check that covers the cross-service boundary; the unit suites cover projection and apply logic but cannot exercise it.
