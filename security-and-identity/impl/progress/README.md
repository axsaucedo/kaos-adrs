# Progress notes

One file per phase, named `P<n>-<slug>.md`. Each note records, for that phase:

- **Status** — not started / in progress / done.
- **PR** — link to the phase PR.
- **Scope delivered** — what was actually built, against the phase scope in [`../../plan/proposed-split.md`](../../plan/proposed-split.md).
- **Validation** — tests run (local and CI) and their results.
- **Follow-ups** — anything deferred to a later phase.

See [`../README.md`](../README.md) for the overall convention.
