## Problem statement

Add the declarative control plane that deploys and operates the M2 memory service: a `MemoryStore` CRD carrying **only infrastructure and models** (no behavioural knobs), an operator controller that reconciles it into the memory-service `Deployment` and `Service` (plus a `PersistentVolumeClaim` in `local` mode), wiring the storage secret and the referenced `ModelAPI` endpoints as environment, validating that model references resolve and are Ready, and reporting health and the service endpoint in status. The inline short-term-only `MemoryConfig` on the Agent is replaced with a **behavioural block** (`enabled`, `type`, `memoryStore`, `scope`, `clientParams`, `tools`, `failureMode`) surfaced to the runtime as the new `MEMORY_*` env. An explicit `type: local|remote` field states the backend intent — `remote` requires a bound `memoryStore` and selects the `RemoteMemory` service backend, `local` forbids one and selects the pod-local short-term `LocalMemory` fallback — and when omitted it is derived from `memoryStore` presence. CRD manifests, deepcopy, RBAC, the memory-service image default, and the Helm chart are regenerated and wired.

## Design plan (how it fits)

- **CRD types.** Add `operator/api/v1alpha1/memorystore_types.go`: `MemoryStoreSpec{ engine (enum mem0), storage{ type (enum local|external), local{ provider (enum chroma), persistentVolume{ size } }, external{ provider (enum pgvector), connectionSecretRef{ name, key } } }, replicas, models{ summarization{ modelAPI, model }, embedding{ modelAPI, model } }, extraction{ concurrency }, defaultFailureMode (enum soft|strict) }` and `MemoryStoreStatus{ phase (Pending|Ready|Failed), ready, endpoint, message, deployment }`, mirroring `ModelAPI`'s markers (shortNames, printcolumns Engine/Storage/Ready/Phase, `subresource:status`). The MemoryStore carries **infrastructure, models, and only the two store-level operational knobs** (`extraction.concurrency`, `defaultFailureMode`); the shared-window/digest/sweeper marks (`token_budget` default, `compaction_*`, `hard_event_cap`, `digest_retention`, `idle_ttl`, `sweep_interval`) stay as memory-service env defaults and are **not** CRD fields in this version. Validation via CEL/markers: `local` requires `local`, `external` requires `external` + `connectionSecretRef`; `replicas==1` enforced for `local`.
- **Controller.** Add `operator/controllers/memorystore_controller.go` modelled on the MCPServer controller: fetch → finalizer → init status → resolve+validate `models.{summarization,embedding}` ModelAPIs (by name, ready-gated, like `agent_controller.go:110-156`) → construct `Deployment` (memory-service image from `DEFAULT_MEMORY_SERVICE_IMAGE`, env from storage config + resolved ModelAPI endpoints + `extraction.concurrency` + `defaultFailureMode`, secret via `valueFrom.secretKeyRef` for `external` DSN) → `Service` → `PersistentVolumeClaim` for `local` → owner refs on all → readiness from the service `/readyz` reflected into status endpoint/phase. RBAC markers for `memorystores`, `persistentvolumeclaims`, plus existing `deployments`/`services`/`configmaps`.
- **Agent block + env redesign.** Replace `MemoryConfig` (`agent_types.go:51-79`) with `MemoryConfig{ enabled, type (enum local|remote, optional), memoryStore, scope (enum private|user|shared|session), clientParams{ tokenBudget, rollingSummary }, tools (enum all|read|write), failureMode (enum soft|strict) }`. `clientParams` is **minimal** by design — only the two knobs the runtime forwards per-agent (`MEMORY_SHORT_TERM_TOKEN_BUDGET`, `MEMORY_ROLLING_SUMMARY`); the shared-window/digest/sweeper/extraction knobs (`compaction*`, `hardEventCap`, `digestMaxTokens`, `idleTtl`, `sweepInterval`, `concurrency`) are service-global and live only on `MemoryStore`, because the window is keyed by scope and shared across agents, so a per-agent override is incoherent. Update `agent_controller.go` to resolve `memoryStore` (same-namespace, name lookup) to a `MemoryStore` endpoint and set the new `MEMORY_*` env M3 consumes (`MEMORY_TYPE`, `MEMORY_STORE_ENDPOINT`, `MEMORY_SCOPE`, `MEMORY_SHORT_TERM_TOKEN_BUDGET`, `MEMORY_ROLLING_SUMMARY`, `MEMORY_TOOLS`, `MEMORY_FAILURE_MODE`, `MEMORY_ENABLED`), removing the old keys and the `local|redis` type. `MEMORY_STORE_ENDPOINT` is injected only when the effective type is `remote`.
  - **Backend switch + explicit type.** The effective backend is `type` when set, else derived from `memoryStore` presence (`remote` if bound, `local` if not). `type: remote` **requires** a `memoryStore`; `type: local` **forbids** one (a `type: local` block with a `memoryStore` set is rejected as a conflict). Same-object field relationships (`type`↔`memoryStore`, `scope`/`tools` requiring a store) are enforced with **CEL validation rules** on the CRD; cross-resource existence/readiness of the referenced `MemoryStore` and `ModelAPI`s is enforced at **reconcile time** via `Blocked`/`InvalidConfig` status conditions.
  - **Fail-closed binding guards.** `scope ∈ {user, shared}` requires a `memoryStore` (CEL); any `tools` setting requires a `memoryStore` (CEL) — `LocalMemory` is pod-local and has no long-term tier, so both would silently no-op otherwise. The fully-qualified `kaos://agent/{namespace}/{name}` identity is always injected as `AGENT_IDENTITY` for private-scope ownership (name-only fallback dropped) so identity-less agents can never collapse onto one shared private partition.
  - **Degraded, not un-Ready.** Memory is an augmentation, never a tier-1 dependency: when the bound `memoryStore` is not Ready the operator surfaces a `MemoryDegraded` **status condition** on the Agent and **keeps the pod Ready** (it serves short-term-only), rather than failing the pod readiness probe or marking the Agent CRD not-Ready — a store outage must not remove the agent from serving.
- **Chart/images.** Add `defaultImages.memoryService` to `values.yaml`, emit `DEFAULT_MEMORY_SERVICE_IMAGE` in `operator-configmap.yaml`, add the `MemoryStore` CRD to `chart/crds/`, and the controller RBAC to `kaos-operator-rbac.yaml`. Register the controller in `main.go`/`SetupWithManager` and the scheme.
